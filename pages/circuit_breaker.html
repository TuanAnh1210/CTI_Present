Runtime Control — Circuit breaker (Feature Store)

State machine bảo vệ Prediction Service khi gọi Feature Store: vượt ngưỡng lỗi → cắt mạch → fast-fail sang fallback/not-scorable. Giám sát qua metric Ch.4 (feature_store_timeout_rate, 5xx).

Máy trạng thái CLOSED → OPEN → HALF-OPEN

Bấm vào trạng thái để xem chi tiết hành vi.

CLOSED

Mạch đóng — gọi Feature Store bình thường. Có timeout và retry với exponential backoff. Đếm error rate trong cửa sổ 1 phút.

OPEN

Error rate vượt ngưỡng (ví dụ >5%/1 phút) → mở mạch. Mọi request FS bị fast-fail ngay, không chờ timeout từng call → chuyển Fallback / Not-Scorable (Lv.2–3).

HALF-OPEN

Sau cooldown 30s, cho ~10% traffic thử gọi FS. Probe OK liên tiếp → đóng mạch (CLOSED). Một request fail → quay OPEN.
